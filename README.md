---

## 📖 Descripción del Proyecto

Este proyecto es una aplicación web estática que presenta el portal de contacto de una empresa tecnológica llamada **ConnectTech**. 

El principal valor técnico de este repositorio radica en su **arquitectura dual**: el diseño ha sido implementado utilizando **Bootstrap 5** (paradigma basado en componentes) y replicado con total exactitud usando **Tailwind CSS** (paradigma *utility-first*). Los usuarios pueden navegar fluidamente entre ambas versiones en tiempo real mediante un botón integrado en la barra de navegación para comparar la estructura del código y el rendimiento.

---

## ✨ Características y Funcionalidades

* 📱 **Diseño 100% Responsivo (Mobile-First):** Adaptabilidad perfecta en dispositivos móviles, tablets y pantallas de escritorio.
* 🔄 **Navegación Cruzada:** Interfaz interactiva que permite alternar instantáneamente entre la versión de Bootstrap y la de Tailwind.
* 🎨 **UI/UX Refinada:** * Formularios estructurados con inputs enriquecidos mediante iconos integrados.
  * Efectos de transición (*hover*) en botones, enlaces y redes sociales.
  * Uso de sombras suaves y bordes redondeados para un aspecto moderno.
* ⚓ **Sticky Footer:** Uso avanzado de Flexbox para mantener el pie de página siempre en la parte inferior de la ventana, independientemente de la cantidad de contenido.

---

## 🛠️ Tecnologías y Herramientas Utilizadas

* **Estructura y Estilos Base:** `HTML5`, `CSS3`
* **Framework 1:** `Bootstrap v5.3` (Implementación mediante archivo local para personalización)
* **Framework 2:** `Tailwind CSS v3` (Implementación vía CDN para desarrollo ágil)
* **Tipografía:** `Google Fonts` (Familia tipográfica *Poppins*)
* **Iconografía:** `Font Awesome v6.5` (Iconos vectoriales sólidos y regulares)

## 🧠 Propósito Técnico del Ejercicio

Este desarrollo sirve como un estudio comparativo directo:
1. **El enfoque de Bootstrap:** Demuestra el dominio de los sistemas de cuadrícula clásicos (`container`, `row`, `col`) y la manipulación de componentes pre-estilizados.
2. **El enfoque de Tailwind:** Evidencia la capacidad de construir interfaces complejas directamente en el marcado HTML mediante clases utilitarias, eliminando la necesidad de cambiar constantemente de contexto entre HTML y CSS, y reduciendo el peso final del código.
